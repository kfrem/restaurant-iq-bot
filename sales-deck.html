Common Questions

We've heard every objection.
Here's the honest answer.

"My staff won't use it."
They already use Telegram. We're not asking them to change behaviour — just send their usual end-of-shift update in the group instead of a personal chat. The ones who try it first become the biggest champions. Usually happens within the first week.
"I'm not technical. I can't set this up."
The entire setup takes 30 minutes. If you can set up a Telegram group, you can set this up. There's no coding. No servers. The guide is written for someone who's never seen code before. We also offer a hands-on setup call.
"I already pay for [other software]."
Does your other software have a group chat where your chef can send a voice note at midnight and it turns into a P&L entry? Probably not. Restaurant-IQ is different — it captures information in the moment, from the floor. Nothing else does this.
"What happens to my data?"
Your data lives on your own server. The AI APIs (Google Gemini, Groq) are ones you control with your own API keys. Nothing is sold or shared. You can export all your data as CSV any time. You can delete everything with one command if you ever leave.
"What if the platform goes down?"
Restaurant IQ runs on Railway, which has 99.9% uptime. If there's ever an issue, the deployment recovers automatically. Your database is stored on a persistent volume — no data is ever lost if the system restarts. We also monitor deployments in real time.
"Is it really free to start?"
Yes. Google Gemini API is free. Railway has a free tier. Telegram is free. SQLite is free. There's no catch. You only pay if you scale to 100+ restaurants or choose to upgrade to a managed service tier. Most single-site owners pay £0 forever.
