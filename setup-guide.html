1. What Is Restaurant IQ?

The big picture in plain English

Restaurant IQ is a chat-based management platform for restaurant owners and their teams. Staff send normal messages — voice notes, photos of invoices, text updates — and the system automatically converts everything into business intelligence.

🎤
Voice Notes

Automatically transcribed and categorised

📸
Invoice Photos

Supplier, total, VAT, line items extracted

📝
Text Messages

Waste, staff issues, revenue updates logged

✅
Result: Weekly AI-generated briefings with P&L summaries, cost alerts, action priorities — as a branded PDF — delivered every Monday morning. No manual spreadsheets. No new apps to learn.
⚡ What it does automatically
  • Transcribes voice notes (Whisper AI)
  • Reads invoice photos & extracts data
  • Tracks revenue, costs, waste, staff issues
  • Alerts you when invoices are overdue
  • Monitors stock below par levels
  • Sends weekly report every Monday 8am
  • Flags UK compliance issues in real time
🏗️ Technical stack
  • Python 3.11+
  • SQLite database (zero server setup)
  • Telegram API (python-telegram-bot)
  • Google Gemini / Groq / Claude AI
  • faster-whisper (voice transcription)
  • ReportLab (PDF generation)
  • Built-in web dashboard (no extra server)
📋

3. Setup Step-by-Step

From zero to a running system — follow in order

🤖 Step 1 — Create Your Telegram Access
1

Open Telegram and search for @BotFather

This is Telegram's official tool for creating Telegram integrations. It's a verified account with a blue tick.

2

Send /newbot to BotFather

Follow the prompts. Choose a name (e.g. "My Restaurant Bot") and a username ending in bot (e.g. myrestaurant_iq_bot).

3

Copy the access token

BotFather will send you a message like:
Use this token to access the HTTP API: 1234567890:ABCdef...
Copy the entire token and save it somewhere safe. This is your TELEGRAM_BOT_TOKEN.

4

Disable privacy mode (IMPORTANT for group chats)

Send /setprivacy to BotFather → select your entry → select Disable. This allows Restaurant IQ to read all messages in a group, not just commands.

📂 Step 2 — Fork the Code to Your GitHub
1

Go to the Restaurant-IQ GitHub repository

Click Fork in the top right. This copies the code to your own GitHub account so you can deploy it from Railway.

2

Your repository is now at github.com/YOUR-USERNAME/restaurant-iq-bot

You own this copy. You can make changes and they will automatically redeploy.

🚂 Step 3 — Deploy to Railway
1

Go to railway.app and sign in with GitHub

2

Click New Project → Deploy from GitHub repo

Select your forked restaurant-iq-bot repository.

3

Add a Volume for persistent database storage

In Railway: go to your service → + Add Volume → mount path: /data. This keeps your database safe when the system restarts.

4

Add all Environment Variables (see Section 4)

Go to your service → Variables tab → add each key/value pair.

5

Click Deploy

Watch the build logs. After 2–3 minutes you should see: Restaurant IQ started successfully.

5. Railway Deployment Details

Production hosting — step by step with screenshots guide

🗺️ Railway Dashboard Walkthrough
1

After deploying, find your service URL

In Railway → your project → your service → Settings tab → scroll to Networking → click Generate Domain. This gives you a URL like yoursite.railway.app. Your Restaurant IQ dashboard will live at yoursite.railway.app/dashboard/TOKEN.

2

Check the deployment logs

Railway → your service → Deployments tab → click the latest deployment → view Build Logs and Deploy Logs. You should see Application startup complete.

3

Verify Restaurant IQ is live

Open Telegram, find your Restaurant IQ entry (by the username you chose), send /start. You should receive a welcome message within 5 seconds.

4

Managing restarts & updates

When you push code changes to GitHub, Railway automatically rebuilds and redeploys. Zero downtime between old and new version. You can also trigger a manual redeploy from the Railway dashboard.

✅ Healthy deployment shows
  • Green "Active" status badge
  • Restaurant IQ started successfully in logs
  • Dashboard running on port XXXX
  • Restaurant IQ replies to /start in Telegram
  • Volume mounted at /data
❌ Common deployment errors
  • TELEGRAM_BOT_TOKEN not set — add the variable in Railway
  • GEMINI_API_KEY missing — add the variable
  • Token conflict — another process using same token
  • Database permission error — check volume is mounted
  • Build fails — check Python version is 3.11+

6. Register a Restaurant

First-time setup for each restaurant location

💡
Each restaurant needs its own Telegram group. Add Restaurant IQ to the group first, then register. One instance can manage unlimited restaurants.
🔗 Step 1 — Add Restaurant IQ to a Telegram group
1

Create a new Telegram group for the restaurant

Name it something obvious like "The Crown Inn — Staff Updates" or "Kitchen Team 🍳".

2

Add Restaurant IQ to the group

In the group → tap the group name → Add Members → search for your Restaurant IQ username → add it. Give it Admin permissions so it can read all messages.

3

In the group, type /register The Crown Inn

Replace "The Crown Inn" with your restaurant name. Restaurant IQ will start a 4-step conversation to collect company details.

📝 Registration conversation — what Restaurant IQ asks
StepRestaurant IQ asks forExample answer
Step 1Legal company nameThe Crown Inn Ltd
Step 2Address, city, postcode15 High Street, Oxford, OX1 1AA
Step 3Phone, email, VAT number (optional)01865 123456 / info@crowninn.co.uk / GB123456789
Step 4Cuisine type & number of coversBritish / 80 covers

You can skip any optional field by typing skip. All details can be updated later with /profile.

👥 Adding staff

Staff are automatically registered the moment they send their first message in the group. No manual setup needed. Restaurant IQ captures their Telegram name and assigns them the role "staff".

✅
Just invite staff to the Telegram group and tell them to start sending voice notes or text updates. That's it.

7. Daily Use — What Staff Should Send

What to capture, and how to send it

🎤 Voice Notes (Best for speed)

Just hold the microphone button in Telegram and speak naturally. Examples:

  • "We did £2,400 today, 78 covers, really busy lunch service"
  • "Jamie called in sick this morning, had to get agency cover"
  • "Fridge 2 is running hot, about 8 degrees, need engineer"
  • "Binned 4kg of salmon, went off before we could use it"
  • "Tips tonight were £180 cash, split between 6 staff"
📸 Invoice Photos (Point and shoot)

Take a photo of any delivery invoice or receipt and send it to the group. Restaurant IQ extracts:

  • Supplier name
  • Invoice date & due date
  • Total amount & VAT
  • Individual line items & prices
  • Payment terms (Net 30, etc.)
📝 Text Messages (Type anything)

Revenue

"Saturday lunch £1,850, 64 covers, good service"

86'd Items

"Ran out of truffle risotto and sea bass tonight"

Staff

"Maria was 20 mins late for dinner service again"

Issues

"Dishwasher making a grinding noise since 3pm"

Waste

"Wasted 2kg chicken thighs and half box of tomatoes"

Compliance

"New salmon from different supplier today, no allergen sheet yet"

⚠️
Important: Restaurant IQ needs to be an Admin in the group to read all messages (not just commands). If staff messages are not being captured, check its admin status in group settings.

9. AI Tiers & API Keys

How Restaurant IQ automatically scales with restaurant count

🟢
Tier 1 — Growth
0–49 restaurants
gemini-2.0-flash

Free Google Gemini API. No cost whatsoever.

🟡
Tier 2 — Scale
50–99 restaurants
llama-3.3-70b

Free Groq API. Fast Llama inference.

🔵
Tier 3 — Enterprise
100+ restaurants
claude-haiku-4-5

Paid Anthropic Claude. Professional quality.

🔄 How tier switching works

Restaurant IQ automatically counts how many restaurants are registered and picks the right AI provider based on that count. You don't need to do anything — just add the API keys for the tiers you need.

💡
If an API key is missing for a tier, the system falls back to the previous tier. So if you only have a Gemini key, all analysis always uses Gemini regardless of restaurant count.
🎤 Voice Transcription — Always Local

Voice notes are transcribed using faster-whisper running locally on your server. No audio is ever sent to a cloud API. The model downloads once on first use and is cached. Use WHISPER_MODEL_SIZE=tiny for speed, base for better accuracy.

10. Web Dashboard

Live business overview — accessible from any browser

🔗 How to access your dashboard
1

In your restaurant's Telegram group, type /dashboard

Restaurant IQ replies with a private link unique to your restaurant.

2

Click the link — dashboard opens in any browser

URL format: https://yoursite.railway.app/dashboard/YOUR-UNIQUE-TOKEN

3

Dashboard auto-refreshes every 60 seconds

Bookmark it. Pin it on a tablet in your back office for a live operations view.

📊 What you see on the dashboard
  • 📅 This week's staff rota (by day)
  • 📦 Stock levels vs par levels (red = low)
  • 🧾 Outstanding invoices by due date
  • 💰 Month-to-date P&L summary
  • 📝 Last 10 activity entries
  • 🚫 Top 10 most 86'd menu items
  • ⚠️ Unresolved allergen alerts
🔐 Dashboard security

Each restaurant's dashboard has a unique random token in the URL. Only people with the link can see the data. To regenerate a new token (if you want to revoke access), contact support or clear the token from the database.

11. UK Compliance Features

Built-in tools for legal requirements — no extra work needed

⚖️ Tips Act 2023

The Employment (Allocation of Tips) Act 2023 requires employers to keep a 3-year record of all tips and their distribution.

  • Restaurant IQ auto-detects tips in any message
  • Logs date, amount, type (cash/card/both), staff count
  • 3-year retention built in (no auto-delete)
  • /tips — view log for any period
  • /tipsreport — formal compliance document
🥜 Natasha's Law (Allergens)

Requires full allergen traceability — especially when suppliers change ingredients.

  • Auto-flags when new supplier mentioned in invoice
  • Auto-flags when allergen concern noted in message
  • /allergens — 90-day traceability log
  • /resolvallergen [id] — mark as reviewed
  • Unresolved alerts shown on dashboard
🔍 FSA Food Hygiene Inspection

/inspection generates a 90-day readiness report covering everything an inspector checks.

  • Supplier traceability records
  • Equipment maintenance logs
  • Temperature incidents flagged
  • Allergen alert history
  • Staff records & training notes
📊 GDPR Data Management

Control your data retention to stay compliant with GDPR.

  • /deletedata 90 — delete entries older than 90 days
  • /cleardata CONFIRM — wipe all entries
  • Staff data is name + Telegram ID only
  • No data shared with third parties (except AI APIs you control)

12. Troubleshooting

Most common problems and how to fix them in under 2 minutes

❓ Restaurant IQ isn't responding to messages
1

Check Restaurant IQ is an Admin in the group

Group settings → Admins → make sure Restaurant IQ is listed with "Read messages" permission.

2

Verify privacy mode is disabled

In BotFather, send /setprivacy → select your entry → confirm it says DISABLED. Then remove and re-add Restaurant IQ to the group.

3

Check Railway deployment is running

Go to Railway → your project → check the service shows a green "Active" badge. If it's crashed, click Redeploy.

Common error messages and fixes
ErrorCauseFix
Not registered Group hasn't been registered yet Type /register RestaurantName
AI analysis failed API key missing or rate limited Check GEMINI_API_KEY in Railway Variables
Voice not transcribed Whisper model still downloading Wait 2–3 minutes, try again
Database locked Two instances running same DB file Ensure only one Railway deployment is active
Dashboard not loading Railway domain not set up Railway → Settings → Networking → Generate Domain
PDF not sending Restaurant IQ doesn't have file permission in group Give Restaurant IQ full Admin permissions
